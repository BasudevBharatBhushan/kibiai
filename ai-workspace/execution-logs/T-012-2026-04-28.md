#### CHATBOT WIRING FIX (Bug corrected in this session)
**Problem:** `formatPrompt` was embedding DB schema INTO the user message, making the user
bubble show the full technical prompt string.

**Root Cause:** `REPORTS_SYSTEM_INSTRUCTION` defines:
- `predefined_prompt` = schema context (TYPE 1/3/4)
- `user_prompt` = clean user intent (TYPE 2)
`responses.ts` concatenates `predefined_prompt + user_prompt` before sending to OpenAI.
Our implementation was putting schema in `user_prompt` via `formatPrompt` instead.

**Fix:**
- `predefinedPrompt` = `buildPredefinedPrompt(state.setup, state.config)` — carries DB schema + date context
  → User bubble stays clean, shows only what the user typed
  → AI receives full context via `predefined_prompt` field in API
- `formatPrompt` = just appends `.json` suffix (TYPE 2 per system instruction)
  → `user_prompt` = clean user intent + ".json"
- Refreshes automatically when setup or config changes (TYPE 3/4 scenarios)
