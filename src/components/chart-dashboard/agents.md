# Module: Chart Dashboard

## Overview
This module handles the visualization of charts and business insights on a draggable grid. It uses `react-grid-layout` for the dashboard layout and `Highcharts` for rendering data visualizations.

## Components
- `DashboardGrid.tsx`: The main container that orchestrates the grid layout and renders individual cards.
- `ChartCard.tsx`: Renders standard charts (Bar, Line, Pie, etc.). Includes a "Minus" button to inactivate the chart.
- `InsightCard.tsx`: Renders structured business insights generated by the AI Insight Assistant.
- `EditPanel.tsx`: Sidebar for manual chart configuration and field mapping.

## State Management
- **DashboardContext**: Central state for active/inactive charts, layout modes, and dataset management.
- **Inactivation Logic**: When a chart is "removed" from the grid, it is actually marked as `inactive` in the `canvas-batch` payload. The backend updates `chart_template_status` to `"Inactive"` to persist this state.

## Logic & Flow
1. **Initial Load**: Fetches chart templates for the report.
2. **Data Processing**: `processData` utility maps raw report rows to chart schemas.
3. **Auto-Save**: Any layout change or visibility toggle triggers a debounced PATCH to `/api/report-templates/[id]/charts/canvas-batch`.
4. **Insight Integration**: Business insights are injected as a special `chart_type: "insight"` and rendered via `InsightCard`.

## Icons & UI
- **Inactivate**: Represented by a `FiMinus` icon.
- **Drag Handle**: Represented by `FiMove`.
- **Mode Toggle**: High-level switch between "Copilot" and "Configure" panels.
