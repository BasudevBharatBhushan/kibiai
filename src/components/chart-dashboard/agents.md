# Module: Chart Dashboard

## Overview
This module handles the visualization of charts and business insights on a draggable grid. It uses `react-grid-layout` for the dashboard layout and `Highcharts` for rendering data visualizations.

## Components
- `DashboardGrid.tsx`: The main container that orchestrates the grid layout and renders individual cards.
- `ChartCard.tsx`: Renders standard charts (Bar, Line, Pie, etc.). Includes a "Minus" button to inactivate the chart. Also contains the "Compare" button (visible in both admin and viewer modes) which triggers `CompareModal`.
- `InsightCard.tsx`: Renders structured business insights generated by the AI Insight Assistant.
- `EditPanel.tsx`: Sidebar for manual chart configuration and field mapping.
- `CompareModal.tsx`: Full-screen comparison modal. Manages the right-panel state machine (PICKING -> LOADING/GENERATING -> VIEWING/NEW_FILTER). Handles both User Mode (report vs report) and Admin Mode (template preview vs report). Renders as a `createPortal` to `document.body` to escape overflow/z-index stacking. The VIEWING state exposes actions for selecting another report or opening the new-filter form without closing the modal.
- `CompareChartPanel.tsx`: Pure presentational component rendering one side of the comparison (header badge + report metadata + Highcharts chart). Handles Insight charts separately by listing `insight_results` as a text list.
- `ReportHistoryPicker.tsx`: Lists saved reports for the same `report_template_id`, filters out the current report, and provides a "Generate with New Filter" CTA.

## State Management
- **DashboardContext**: Central state for active/inactive charts, layout modes, and dataset management.
- **Inactivation Logic**: When a chart is "removed" from the grid, it is actually marked as `inactive` in the `canvas-batch` payload. The backend updates `chart_template_status` to `"Inactive"` to persist this state.
- **CompareModal State**: The modal uses a local right-panel state machine (`RightPanelState`) to control what is shown on the comparison side. It reads `isViewerMode` and `templateId` from `DashboardContext` to determine operating mode.

## Logic & Flow
1. **Initial Load**: Fetches chart templates for the report.
2. **Data Processing**: `processData` utility maps raw report rows to chart schemas.
3. **Auto-Save**: Any layout change or visibility toggle triggers a debounced PATCH to `/api/report-templates/[id]/charts/canvas-batch`.
4. **Insight Integration**: Business insights are injected as a special `chart_type: "insight"` and rendered via `InsightCard`.

## Chart Comparison Feature (T-033)
### User Mode (Viewer — `/reports/[id]/charts`)
- Left panel = current report's `ChartConfig` (already processed by `DashboardContext`)
- Right panel = user picks from history OR generates a new report
- New report is fetched via `GET /api/reports/[id]`, schema matched by `pKey === primaryConfig.id`, then `processData()` is called

### Admin Mode (Analyzer — `/templates/[id]/charts`)
- Left panel = same chart config from the template's preview dataset
- Right panel = same picker / new filter flow, using the template's `report_template_id`
- Template metadata fetched from `GET /api/report-templates/[id]/charts` on modal mount

### Schema Matching Rule
The comparison always finds `schemas.find(s => s.pKey === primaryConfig.id)` in the comparison report. If not found, a clear error is shown and the user is returned to the PICKING state.

## Icons & UI
- **Inactivate**: Represented by a `FiMinus` icon.
- **Drag Handle**: Represented by `FiMove`.
- **Compare**: Represented by `FiGitMerge`. Opens `CompareModal`.
- **Mode Toggle**: High-level switch between "Copilot" and "Configure" panels.
